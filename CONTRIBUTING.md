# 参与贡献指南

感谢你对 Project GreenEdge 的关注！这里会告诉你如何参与。

## 5 分钟快速开始

### 对 GitHub 不熟？

没关系，你可以：

- **直接反馈**：到 [Issues](https://github.com/Yuye0325/project-greenedge/issues) 页面提出想法或建议
- **邮件参与**：把你的代码/想法发给项目维护者

### 熟悉 GitHub？

1. Fork 本仓库
2. 创建你的特性分支：`git checkout -b feature/你的功能`
3. 提交改动：`git commit -m '添加了什么'`
4. 推送到分支：`git push origin feature/你的功能`
5. 提交 Pull Request

## 参与方向

### 数据科学
- 完成建筑用电预测挑战（目标 MAPE < 5%）
- 提出新的挑战任务方向
- 贡献数据集或特征工程方案

### 前端开发
- 改进项目主页设计
- 添加数据可视化组件
- 优化移动端体验

### 后端/DevOps
- 完善 CI/CD 流程
- 搭建数据管道
- 自动化测试

### 文档/翻译
- 翻译文档为英文或其他语言
- 撰写教程和使用指南
- 整理学术参考文献

### 传播推广
- 转发给环保组织和技术社区
- 在论坛/社交媒体上介绍项目
- 邀请朋友一起参与

## 贡献者墙

每当你提交的 PR 被合并，你的 GitHub 用户名会自动出现在项目主页的贡献者墙上。

## 行为准则

- 保持友善和尊重
- 接受建设性批评
- 聚焦于对社区最有利的事情
- 对不同观点和经验表示包容
