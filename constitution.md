# Project GreenEdge（绿刃计划）透明宪章

> 本文档是 Project GreenEdge 的根本性治理文件。所有贡献者、协作者和使用者均受本宪章约束。

---

## 第一条：公益非个人所有原则

1. Project GreenEdge（以下简称"本项目"）是一个**公益开源项目**，不属于任何个人、公司或组织。
2. 本项目属于所有贡献者和全人类。任何个人不得将项目或其资产据为己有。
3. 项目的 Gitee 仓库、域名、收款账户等资源均为项目共有资产，管理者仅作为临时受托人，不得用于个人目的。
4. 若项目因故终止，所有剩余资产（包括但不限于代码、数据、资金）将转为公共领域或捐赠给官方认可的公益机构。

---

## 第二条：财务完全透明原则

1. 本项目的一切收入与支出，必须在发生后 **48 小时** 内在 `finance/ledger.md` 中公示。
2. 每笔收支记录必须包含：日期、金额、事由、凭证链接、经手人。
3. 所有收款必须进入项目专用账户，与个人账户严格隔离。
4. 任何单笔超过 500 元人民币（或等值外币）的支出，须在 Gitee Issues 中提前公示并获得至少 2 位贡献者的认可。
5. 每月进行一次财务结余公示，接受社区任意成员审计。

---

## 第三条：贡献公开原则

1. 本项目所有代码、模型、文档、数据的贡献过程均在 Gitee 仓库公开可查。
2. 每位代码/文档贡献者均应在 `CONTRIBUTORS.md` 中留名。
3. 鼓励贡献者留下"减碳承诺"，但非强制。
4. Pull Request 审核以代码质量和项目目标为准，不因贡献者的国籍、性别、经验水平而区别对待。

---

## 第四条：协作友善原则

1. 尊重所有背景、所有经验水平的贡献者。
2. 禁止任何形式的骚扰、歧视、人身攻击。
3. 建设性批评优于否定性批评。当你不认同某个方案时，请提出替代建议。
4. 鼓励老手指导新手，知识传递是本项目的核心价值之一。

---

## 第五条：宪章修订

1. 本宪章的修订须通过 Pull Request 提出，并在 Gitee Issues 中公示至少 7 天。
2. 修订需获得至少 3 位非关联贡献者的赞同票方可生效。
3. 第一条（公益非个人所有原则）和第二条（财务完全透明原则）不可修订或废止。

---
